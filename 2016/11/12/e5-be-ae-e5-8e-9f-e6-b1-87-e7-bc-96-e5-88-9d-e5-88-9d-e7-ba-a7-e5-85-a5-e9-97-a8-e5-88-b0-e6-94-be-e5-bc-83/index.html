一、基本指令

  1. mov dst, src

    关于mov，没什么好说的，只是注意一点  

mov \[ax\], \[bx\]  

这样的语句是错误的，mov语句的dst和src不能同时为储存单元的值

所以正确使用姿势是  

mov al, \[si\]  

甚至是  

mov al, \[si + 1\]

2.lea dst, src

取有效地址到dst中  

如： 你在data段（文件开头定义那里）上，定义了一个buffer  

如：  

data segment  

    buffer db 100  

              db  0  

              db  dup(0)  

ends  

此时，使用lea si, buffer ,那么si里就存下了buffer的偏移地址

3.cmp dst, src

最最重要的一个指令  

（1）若dst 与 src相等， 则置ZF=1  

（2）CF，OF视情况而定  

        当是无符号数比较时，  

            如果 CF = 0, ZF = 0 则 (dst) > (src)  

            如果CF = 1               则(dst) <(src)  

        当是有符号数比较时  

            如果OF = 1 则运算溢出  

            如果OF∨SF = 0, (dst) > (src)  

            如果OF∨SF = 0, (dst) <(src)

cmp最大的作用是进行比较后跳转，详见后文分支技术

4. mul和div

 我老是记不住使用方法的指令（cmp也是）  

 mul src  

 div src

(这里只列出无符号数的乘除)  

 这指令有隐含目的操作数，所以需要先赋值为被除数  

 mov AX, 12345  

 mov BL  

 div BL  

 需要注意的是  

 mul 当src为字节时（八位），需要先给AL赋乘数，结果保存在AX  

        当src为字时（十六位），需要先给AX赋值，结果保存在DX:AX   

  div 当src为字节时， 需要给AX赋值， 表示**16位的AX**除以8位的src，商存在AL，余数存在AH  

        当src为字时， 需要给DX,AX赋值,表示32位的除以16位的src,商存在AX,余数存在DX  

 **更需要注意的是，**

如果你想要8位乘以8位，即使乘完的结果没有大于8位（不超过AL的存储大小），但是还是会影响到AH的值（因为结果存到AX里）

如果你想要8位的除以8位时，会用到  

 mov AL, \[si\]  

 此时需要看看AH位是否为0，不然会出不期望的结果  



 5\. push pop  

     push pop 注意顺序  

     如：  

     push AX  

     push BX  

     那么pop时，应为  

     pop BX  

     pop AX

二、寄存器与存储单元

寄存器和存储单元的区别，一个是芯片内部存数据的地方，一个是外设  

寄存器：  

   AX: 通常用作计算，AH用作中断时读取的终端号，AL用作调用函数时传递参数  

   BX: 存放地址（因为基本小代码基本只有一个数据段，所以也没怎么用着，就随便用了）  

   CX: loop循环时的计数呀  

   DX:放数据的  

   SI  :存放源操作数的段内偏移地址  

   DI : 存放目的操作数的段内偏移地址  

    （SP,BP就不提了）  

    关于SI,DI 通常使用方式为，假设任务是 把存在一个存储单元的值取出来，然后经过转化后，存到另一个存储单元上去  

    于是代码为：  

    lea si, buff1  

    lea di, buff2 ; 从buff1的值转换后存到buff2中  

    mov dl, \[si\] ; 取出si指向的存储单元的值给dl（假设是8位的数据）  

    balabala~~一堆转换  

    mov \[di\], dl; 将值存到di指向的存储单元  

存储单元：  

    经过上例，也可以明白，当使用\[si\]时，取的是存储单元的值，若直接si，则是修改si的值  

    mov al, \[si\] ;取的是si指向的存储单元的值  

    mov al, si   ;取的是si的值  

另外常用的是si, di的自增自减  

    inc si  

    dec si

三、函数

函数是我们大大节省代码量，提高代码重用率的手段!  

(这里我们先只介绍段内的短跳转调用)  

调用一个函数  

call FunctionName  

定义函数：  

FunctionName proc near   

        函数体  

        ret  

FunctionName endp  

关于函数，要注意的是，保护寄存器上的值，所以进入子函数后应push要保护的数据，退出子函数前pop回来  

关于入口参数，通常使用AX进行传递需要处理的数据，SI用来传递偏移地址

四、循环分支技术

  关于汇编最麻烦的地方往往是循环和判断，因为常常感觉满屏都是C中的goto语句既视感，所以感觉思维方式与写C时大相径庭  

**  循环：**  

  先要给CX赋循环次数，然后往后运行时，每一次遇到LOOP语句，CX都会自减1  

  当用到嵌套循环时，先要push CX将外层循环的值存起来，再使用CX

  以下列出C中的循环，在汇编的实现方式

关于cmp语句的补充：

当if语句的条件成立时，通过哪个语句触发跳转到label处

cmp a, b // CF = 0

五、附加说明

关于定义一个keybuff时的格式

data segment

keybuff db 100  

             db 0  

             db 100 dup(0)

ends

这种格式的定义，通常用于字符串输入的时候使用

lea dx, keybuff

mov ah, 0Ah

int 21h

此时，输入的数据会存入 db 100 dup(0)处，所以如果你要取首个输入的数据时，应

lea si, keybuff

mov dl, [si + 2]

或

lea si, keybuff + 2

mov dl, [si]

那么定义的时候第二位db 0，将会被修改为输入字符的个数

如：若输入

Hello world!$

加上下划线更好体现个数为13，那么第二位的值自动修改为13

（关于以上字符串后面加$号，倘若只是输入数据进行处理，并不需要$。假如的确是字符串的，则需要加上$）

关于输入与输出显示的：

在输入框中输入的都是ASCⅡ码，若是数字，需要减去30h才是实值。

倘若输入的是数字：74

那么就占用两个存储单元放着37h,34h需要减去30h才是实值7，4

若这74是两位数的意思，则需要再进一步处理7*10+4咯

而输出时，则是需要转成ASCⅡ码

若是两位数，则要进行以上所述的逆操作

ASCII码表对应的数值是十进制的

汇编语言从键盘中输入的是字符，读进去的时候是ASCII码

调试软件里面显示的是16进制的

为方便开发，加快开发效率，给出几个子函数，不过请不要无脑CC and CV

1. 输入的数组字符串ASCⅡ转换成十进制

输入的数组字符串，数字与数字间以‘，’隔开，末尾没有

databuff为输入的数组字符串（即上文的keybuff），num用于存放从ASCⅡ转换成十进制后的数字（这不是一段子函数）

  1. mov bl, 44 ; bl
  2. mov bh, 10 ;bh
  3. lea si, databuff + 2
  4. lea di, num + 1
  5. xor cx, cx ;cx
  6. mov cl, [si - 1]
  7. mov dh, 1
  8. deal:
  9. cmp [si], bl
  10. jnz continue
  11. inc di
  12. inc si
  13. inc dh
  14. loop deal
  15. continue:
  16. mov al, [di] ;al
  17. mul bh
  18. mov [di], al
  19. mov dl, [si]
  20. sub dl, 48
  21. add [di], dl
  22. inc si
  23. loop deal
  24. lea si, num
  25. mov [si], dh

2. 十进制转换为ASCⅡ码

输入为AL，即要转换的值

需要在data段中开辟一个3位的空间，格式为

  1. buff db 0
  2. db 3 dup(0)

第一位db 0 用于储存转换后数字占的位数

  1. ;---------------hex2asc-------------------
  2. ;input AL
  3. ;output buff
  4. ;-----------------------------------------
  5. transhex2asc proc near
  6. push di
  7. push bx
  8. xor ah, ah
  9. lea di, buff + 1 ;di
  10. mov bh, 10 ;bh
  11. mov bl, 0 ;bl
  12. cmp al, 99
  13. ja cx3
  14. cmp al, 9
  15. ja cx2
  16. mov cx, 1
  17. jmp trans
  18. cx2:
  19. mov cx, 2
  20. jmp trans
  21. cx3:
  22. mov cx, 3
  23. trans:
  24. div bh
  25. mov [di], ah
  26. inc di
  27. inc bl
  28. xor ah, ah
  29. loop trans
  30. lea di, buff
  31. mov [di], bl
  32. pop bx
  33. pop di
  34. ret
  35. transhex2asc endp

3. 将一个实值（即如果数字是31，存在储存单元中的值也是31）以十进制打印出来

输入为AL，即要转换的值

依赖子函数第二条

  1. ;------------dispscores-----------
  2. ;input al
  3. ;---------------------------------
  4. dispsc proc near
  5. push si
  6. push cx
  7. push ax
  8. push dx
  9. call transhex2asc
  10. lea si, buff
  11. xor cx, cx
  12. mov cl, [si]
  13. inc si
  14. add si, cx
  15. sub si, 1
  16. mov ah, 02h
  17. dis:
  18. mov dl, [si]
  19. dec si
  20. add dl, 30h
  21. int 21h
  22. loop dis
  23. pop dx
  24. pop ax
  25. pop cx
  26. pop si
  27. ret
  28. dispsc endp

4. 打印一个回车键

  1. ;------------display \n-----------
  2. dispcr proc near
  3. push ax
  4. push dx
  5. mov ah, 2
  6. mov dl, 0ah
  7. int 21h
  8. mov dl, 0dh
  9. int 21h
  10. pop dx
  11. pop ax
  12. ret
  13. dispcr endp
